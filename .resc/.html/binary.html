Le binaire c'est quoi ?

Une introduction simple aux notions de binaire, bit et octet.

Le binaire

Le binaire est un système de numération avec deux chiffres (0, 1) [base 2] contrairement au système décimal [base 10] que nous utilisons habituellement, composé de dix chiffres (0 à 9).

Bit et octet

Le bit est la plus petite unité de stockage numérique : il peut valoir 0 ou 1. Un seul bit suffit pour représenter un état actif/inactif (vrai/faux), comme une case à cocher. Plusieurs bits combinés servent à représenter des données plus complexes (texte, images, son, vidéo...).

Un octet est une unité standard composée de 8 bits. Avec 8 bits on peut représenter 256 valeurs différentes (de 0 à 255) car il y a 2⁸ = 256 combinaisons possibles.

# Plusieurs exemples de conversion binaire -> décimal

Comment convertir en décimal

Pour convertir un nombre binaire en décimal, on additionne les puissances de deux (2ⁿ) correspondant aux bits positionnés à 1.

  1. Le premier bit (à droite) vaut 2⁰ = 1
  2. Le deuxième vaut 2¹ = 2
  3. Le troisième vaut 2² = 4
  4. Le quatrième vaut 2³ = 8
  5. Le cinquième vaut 2⁴ = 16
  6. Le sixième vaut 2⁵ = 32
  7. Le septième vaut 2⁶ = 64
  8. Le huitième (à gauche) vaut 2⁷ = 128

Exemple

Pour 10111011 :

10111011 -> 128 + 32 + 16 + 8 + 2 + 1 = 187

Note : Un pixel est composé de trois octets : un pour le rouge, un pour le vert et un pour le bleu. Plusieurs de ces pixels forment des images, mais cela peut rapidement devenir très lourd en termes de taille. C'est pourquoi on utilise souvent des méthodes de compression comme le ".png" (sans perte) et le ".jpeg" (avec perte), à l'exception du format bitmap ".bmp" et d'autres formats (qui eux pèsent une tonne). Pour les vidéos, on utilise d'autres méthodes de compression qui tiennent compte des images précédentes et suivantes et bien plus encore.
